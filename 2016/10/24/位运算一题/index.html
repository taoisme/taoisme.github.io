位运算一题

By Tao

Published 2016-10-24

Contents

  4. 4. 解答

背景

近日看到漫画算法：找出缺失的整数这个帖子。其中一个问题是:

一个无序数组里有若干个正整数，范围从1到100，其中98个整数都出现了偶数次，只有两个整数出现了奇数次（比如1,1,2,2,3,4,5,5），如何找到这个出现奇数次的整数？

在此试将其扩展为更一般的情况。

题目

有K（K >= 2）个两两不相同的32位无符号整数，每个数字重复若干次（至少一次），组成一个数组。已知其中K-2个数出现次数为3的倍数。其余两个数，一个出现次数为3的倍数+1，另一个出现次数为3的倍数+2。试求一个最优的算法确定出两个出现次数不为3的倍数的数。

解题思路

位运算通常是指AND, XOR, NOT, SHIFT等对于二进制位表示的一元或者二元运算。从原贴来看，我认为此类位运算问题最根本的解题思路是“分治法”。通过使用位运算，把未知的信息分离为一个个独立的比特位，再通过在每一位上面求解，最后综合起来得出问题的解。

此题的难点在于给的数组中，K-2个数字出现的次数并非2的倍数，而是3的倍数。在此设定下，不能直接通过异或运算消去这些数。但是，我们需要注意到，如果把所有的数都表示成32位二进制，则对于每一比特位(31…0)而言，该位上数字之和模3只有3种情况。我们需要打开思路，在一般位运算的基础上引入模3加法，然后根据结果进行分类讨论。

解答

不失一般性，我们假设A mod 3 = 1, B mod 3 = 2. 首先，我们对于这个数组的每一位单独求和，之后取模3。每一位上的结果有3种情况：
情况一： 结果为0。说明这一位上A和B均为零或者均非零。
情况二： 结果为1。说明这A在这位非零，B在这位为零。
情况三： 结果为2。说明这B在这位非零，A在这位为零。

容易看出，情况二和情况三在32位中至少出现一次，否则有A=B，与题意不合。

接下来以情况二出现为例。假设第i位结果为1。我们以第i位的值为0或1将数组一分为二：一部分包含A（i位值为1），一部分包含B（i位值为0）。

那么问题转化为：
1). 如何在一个由若干个出现3的倍数次的数和一个出现3的倍数+1次的数组中，找出出现次数为3k+1的那一个数。
2). 如何在一个由若干个出现3的倍数次的数和一个出现3的倍数+2次的数组中，找出出现次数为3k+2的那一个数。

对于1)而言，思路就比较明朗了，在每一比特位上数出非零数字出现次数，取模3，最终结果就是出现次数为3k+1的那个数。同理可以得到2)的解。

现在的问题转化为，怎么数每一位上出现的次数？一个简单的做法是我们设置一个计数器 ct[i]，对于32位数的第i位（0 <= i <= 31）。遍历长度为n的数组，每次通过
ct[i] += (a >> i) & 1;
的操作来累加a在第i位的值。最后对于ct的每一位取模3。

这个方法的问题在于，对于n很大的时候counter有可能会存在integer overflow。因此累加应该直接取模3:
ct[i] += ((a >> i) & 1) % 3;
这样解决问题的时间复杂度为O(n)，空间复杂度为O(1)。

在数量级上，上述算法已经为最优了。但我们还可以更加优化复杂度系数的常数项。上面算法保存计数器的时候，需要用到32 Byte的数组。我们继续将其优化到12 Byte。思路是用两个32-bit(4-Byte)的寄存器 (m,n) 来放置模3的信息，另一个32-bit的寄存器 t 作为临时变量。算法如下:

m = 0; n = 0; // m表示2, n表示1, 模3值为2m+n
for (int i = 0; i < n; i++}
{
    x = arr[i]; 
    m = m ^ (n & x); // 处理进位
    n = n ^ x;       // 处理个位数
    /* 若(m, n) = (1, 1),即加法结果为3，将其改为0 */
    t = m & n; 
    m = m ^ t; 
    n = m ^ t; 
}

通过这个小技巧，我们便能节省20 Bytes的存储空间，并且时间复杂度上也有所提高，因为上述算法是在32位上同时进行。
